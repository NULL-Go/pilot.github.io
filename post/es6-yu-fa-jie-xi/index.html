【ES6】语法简学

2021-03-12 7 min read

let 和 const命令

  1. 块级作用域和函数声明

    ES5规定，函数只能在顶层作用域和函数作用域之中声明，不能在块级作用域声明。

    但是，浏览器没有遵守这个规定，为了兼容以前的旧代码，还是支持在块级作用域之中声明函数，不会报错。

    ES6中明确规定，函数在块级作用域中声明类似于let，在块级作用域外不可见。

    但是如果浏览器都明确遵守ES6中的函数声明，那必然会对老代码产生巨大的影响，所以浏览器(其他环境遵循ES6)有以下三条规则:

允许在块级作用域内声明函数。

函数声明类似于var，即会提升到全局作用域或函数作用域的头部。

同时，函数声明还会提升到所在的块级作用域的头部。

  1. const

const声明的同时必须初始化，仅在块级作用域中有效，存在暂时性死区，不可重复声明。

const表示值不能发生改变，const声明的变量若等于简单类型数据，指变量的值不能发生改变，若等于引用类型数据，指变量指向的地址不能改变，但地址指向的数据结构可以改变。

  1. 顶层对象的属性

    顶层对象，在浏览器环境指的是window对象，在 Node 指的是global对象。ES5 之中，顶层对象的属性与全局变量是等价的。(var 和 function定义的全局变量会成为顶层对象的属性)。顶层对象是一个有实体含义的对象，也是不合适的。

    ES6 为了改变这一点，一方面规定，为了保持兼容性，var命令和function命令声明的全局变量，依旧是顶层对象的属性；另一方面规定，let命令、const命令、class命令声明的全局变量，不属于顶层对象的属性。也就是说，从 ES6 开始，全局变量将逐步与顶层对象的属性脱钩。

  2. globleThis对象

    JavaScript 语言存在一个顶层对象，它提供全局环境（即全局作用域），所有代码都是在这个环境中运行。

变量解构赋值

ES6 允许按照一定模式，从数组和对象中提取值，对变量进行赋值，这被称为解构（Destructuring）。

数组的解构赋值

  1. **模式匹配原则：**只要等号两边的模式相同，左边的变量就会被赋予对应的值。
  2. **解构不成功：**变量的值就等于undefined。
  3. **解构报错：**等号右边的值，要么转为对象以后不具备 Iterator 接口，要么是对象但本身就不具备 Iterator 接口。(Generator产生的函数，map，set等等)。
  4. 可以指定默认值：左侧可以指定默认值，但右侧全等于 === undefined时，取默认值。当默认值为表达式时，只有在默认值要起效果时，表达式才会求解出来当做默认值（惰性求值，提高效率）。
// Object.entries 可以与解构赋值一起使用，遍历对象的键值对
let user = {
  name: "John",
  age: 30
};
// 循环遍历键—值对 Object.entries(user): [["name","John"],["age",30]]
for (let [key, value] of Object.entries(user)) {
  alert(`${key}:${value}`); // name:John, then age:30
}

对象的解构赋值

  1. 模式匹配原则：与数组解构的次序不同，属性名匹配。

  2. **实质：**对象的解构赋值的实质是对象的简写，如下是对象结构赋值的实质。解构不成功变量的值也等于undefined。

let { foo: baz } = { foo: 'aaa', bar: 'bbb' }; 
baz // "aaa" 真正被赋值的是**后者**而**不是前者**
let { foo }  <-> let { foo : foo }

​ 真正被赋值的是后者而不是前者，前者是模式匹配

  1. 继承属性的赋值：
function a() {}
a.prototype.name = "father";
let a_ = new a();
let { name: b } = a_;
console.log(b); // father
  1. **可以指定默认值：**类似于数组。
let { a = 1 } = { a: undefined };
// <-> 等价于 let { a: a = 1 } = { a: undefined };
console.log(a);

注意点

  • 如果要将一个已经声明的变量用于解构赋值，必须非常小心。
// 错误的写法
let x;
{x} = {x: 1};
// SyntaxError: syntax error

上面代码的写法会报错，因为 JavaScript 引擎会将{x}理解成一个代码块，从而发生语法错误。只有不将大括号写在行首，避免 JavaScript 将其解释为代码块，才能解决这个问题。

// 正确的写法
let x;
({x} = {x: 1});

  • **字符串，数值和布尔类型的值：**进行解构赋值的时候，会先转换成包装对象。只要等号右边的值不是对象或数组，就先将其转为对象。由于undefined和null无法转为对象，所以对它们进行解构赋值，都会报错。

  • 函数参数的解构赋值：当调用函数传入实参时，相当于形参等于实参进行解构赋值，且可以同时配合函数参数的默认值，当不传入参数或参数为undefined时，形参等于默认参数进行解构赋值。

// 区分下列两种不同的写法
function move({ x = 0, y = 0 } = {}) {
  console.log([x, y]);
}
function move({ x, y } = { x: 0, y: 0 }) {
  console.log([x, y]);
}
// 无论是函数参数的默认值还是解构赋值的默认值，为空或者undefined的时才起作用。
move({ x: 3, y: 8 }); // [3, 8] [3, 8]
move({ x: 3 }); // [3, 0] [3, undefined]
move({}); // [0, 0] [undefined, undefined]
move(); // [0, 0] [0, 0]

Symbol

基础知识

  • Symbol 类型的值直接由函数 Symbol() 生成，用 new Symbol()时将会报错，因为生成的是基本类型的值symbol，而不是像生成对象一样new 出来的。
  • 如果Symbol() 的参数是一个对象，则将对象先toString()转化为字符串 ----> [object Object]。
  • Symbol 可以显式的转化为字符串，但是不是隐式的转化为字符串。Symbol 值也可以转为布尔值，但是不能转为数值。
let sym = Symbol("111");
console.log(222 + sym); // 报错，TypeError: Cannot convert a Symbol value to a number
if (sym) {
  console.log(11); // 可行
}
console.log("aaa" + sym); // 报错，TypeError: Cannot convert a Symbol value to a string
console.log(sym.toString()); // 打印 Symbol(111);

Symbol.prototype.description

ES2019提供了一个实例属性description，直接返回 Symbol 的描述。

let a = Symbol('1'); console.log(a.description); // 返回 '1' ，而不是 a.toString() 的 'Symbol(1)'
